# Conversational Input UI Research

## Direction
Blend these patterns:
- **Typeform/Tally** for one-question-at-a-time focus
- **Intercom / AI chat** for familiar bubble-based interaction
- **Linear / Superhuman** for precision spacing, premium polish, and understated motion
- **Apple-style onboarding** for calm entry and clean completion

## Design rules
1. One active question at a time.
2. Big tap targets, minimal typing.
3. Fixed bottom composer or quick-reply area.
4. Previous answers remain visible as a clean transcript.
5. Progress is visible but not loud.
6. Copy should sound like a sharp team lead, not a goofy chatbot.
7. End-of-day completion should feel satisfying and fast.

## Best fit for this product
This should not be a literal chat app and it should not be a boring form pretending to be chat.

The right answer is a **guided conversational flow**:
- name selection first
- personalized greeting
- assigned questions shown one at a time
- answer captured with either quick replies or one focused input
- friendly closeout

## Anti-patterns
- giant enterprise form screens
- too many visible fields at once
- sluggish fake typing animations
- heavy chrome and dashboard clutter
- tiny low-contrast buttons
- too much color or decoration

## What this prototype is proving
The prototype is validating the **input wedge** before wiring the backend:
- employees can select themselves instantly
- answering assigned questions feels fast
- the UI looks premium enough to drive adoption
- the front-end can later connect cleanly to Apps Script for persistence and orchestration
